1. Scope

Add a temporary debug panel on the Module page that shows:

For each modality (In-Person, Blended, Online, HyFlex)

The contribution of each dimension to the final score

This is read-only, developer-facing, and can be hidden or removed later.

No changes are required to:

navigation (Home / Module / Programme / Visualisations)

Panel A or Panel B inputs

Panel C charts and recommendation

2. Update the scoring function to return a breakdown

Wherever the main scoring function lives (for example scoreModality(inputs) in JS or Python):

Do not change the current final scores.
Keep the existing logic for:

dimensions

weights

final scores per modality

Add a second layer of structured output that returns, for each modality, the dimension-level contribution before weighting.

Example shape, adjust names to match your code:

{
  dimensions: {
    learningDesign: {
      inPerson: number,
      blended: number,
      online: number,
      hyflex: number
    },
    assessmentFit: {
      inPerson: number,
      blended: number,
      online: number,
      hyflex: number
    },
    learnerProfile: {
      inPerson: number,
      blended: number,
      online: number,
      hyflex: number
    },
    stageFit: {
      inPerson: number,
      blended: number,
      online: number,
      hyflex: number
    },
    feasibility: {
      inPerson: number,
      blended: number,
      online: number,
      hyflex: number
    }
  },
  overall: {
    inPerson: number,
    blended: number,
    online: number,
    hyflex: number
  },
  recommended: "inPerson" | "blended" | "online" | "hyflex"
}


If you already have something similar, just ensure it is exposed to the front end.

3. Add a “Debug: Score Breakdown” panel to the Module page

In the Module template (for example module.html):

Under Panel C (where the recommendation and charts are), add a new collapsible section:

Title:
“Debug: score breakdown (temporary)”

Subtitle (small, grey text):

“For diagnostic use when calibrating the scoring model. Not visible in final user version.”

Default state: collapsed.
The user must click to expand.

5. Wiring data to the debug panel

In the front-end JS:

After calling the scoring function, store both:

overall scores

dimensions breakdown

Use the dimensions object to populate the debug tables.

For example:

const scores = scoreModality(inputs);

// Existing: use scores.overall and scores.recommended for charts and summary

// New: use scores.dimensions.learningDesign.online etc. to fill the debug panel


No re-computation should occur in the template. The template should only display the values passed from JS.
